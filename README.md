# AirREGI 原材料在庫管理システム

カフェ向けの原材料ベースの在庫管理システムです。売上データから自動的に原材料在庫を減算します。

## 主な機能

- 🧪 **原材料管理**: 原材料の在庫数をリアルタイムで管理
- 📝 **原料登録**: 商品ごとに使用する原材料と使用量を登録（手動入力対応）
- 📊 **売上連携**: AirREGIの売上データから自動的に原材料在庫を減算
- ⚠️ **在庫アラート**: 原材料が少なくなったら自動的に通知
- 📋 **未登録商品管理**: レシピ未登録の商品を一覧表示
- 📈 **統計情報**: 在庫価値、低在庫原材料数などの統計を表示
- 🔥 **Firebase連携**: リアルタイムデータベースで複数端末から利用可能
- 🎨 **ダークモード対応**: 目に優しいダークモード

## 技術スタック

- **フロントエンド**: SvelteKit 2 + Svelte 5 + TypeScript
- **データベース**: Firebase Firestore
- **ビルドツール**: Vite 5
- **スタイリング**: Tailwind CSS
- **アイコン**: Lucide Svelte

## セットアップ

### 1. リポジトリのクローンと依存関係のインストール

```bash
git clone <repository-url>
cd airregi-inventory
npm install
```

### 2. Firebase のセットアップ

Firebase の設定が必要です。詳細は [FIREBASE_SETUP.md](./FIREBASE_SETUP.md) を参照してください。

1. Firebase プロジェクトを作成
2. Firestore データベースを作成
3. 環境変数を設定（`.env` ファイルを作成）

```bash
cp .env.example .env
# .env ファイルに Firebase の設定を記入
```

### 3. 開発サーバーの起動

```bash
npm run dev
```

ブラウザで `http://localhost:5173` を開いてください。

## 使い方

### 1. 原材料の登録

1. ホームページの「原材料を追加」ボタンをクリック
2. 原材料名、単位、在庫数、発注レベルなどを入力
3. 「追加」をクリック

### 2. 原料（レシピ）の登録

1. 「原料管理」ページに移動
2. 「原料を追加」ボタンをクリック
3. 商品名と使用する原材料を入力
   - 原材料名は直接入力（既存の原材料から自動補完）
   - 使用量と単位を入力
4. 「保存」をクリック

### 3. 売上データのインポート

1. ホームページの「売上取込」ボタンをクリック
2. AirREGIの売上CSVをアップロード（Shift-JIS/UTF-8対応）
3. 自動的に:
   - **登録済み商品**: 原材料の在庫が減算されます
   - **未登録商品**: 未登録商品ページに追加されます
   - **カテゴリ**: 「男性」「女性」などのカテゴリは自動的に無視されます

### 4. 未登録商品の対応

1. 「未登録商品」ページで未登録の商品を確認
2. 「原料登録」ボタンをクリック
3. 自動的に商品名が入力された状態で原料登録画面が開きます

## データベース構造

### ingredients (原材料)

- 原材料名、単位、在庫数、最小在庫レベル
- 仕入先、単価、説明

### recipes (原料/レシピ)

- 商品名
- 使用する原材料リスト（原材料名、使用量、単位）

### unregisteredProducts (未登録商品)

- 商品名（ドキュメントID）
- 累計販売数、初回検出日時、最終検出日時

### sales (売上データ)

- 商品名、カテゴリ、売上額、粗利、販売個数など
